# Task 4 Report — Mountable FabriqAdmin Shell + Internal Router + Plugin Registry

## Summary

Three new files added to `packages/admin-sdk/src/`:

### `router.ts`
- `matchRoute(routes, path)` — splits both route pattern and incoming path on `/`, handles empty-string root, matches static segments exactly, captures one-level `:param` segments into a `params` map. First match wins. Returns `{ route, params }` or `null`.
- `useInternalRouter(initialPath?, basePath?)` — pure `useState`-backed router returning `{ path, navigate, basePath }`. No History API, no `window`/`document` access at module scope. SSR-safe.

### `registry.ts`
- `PluginRegistry` class: `register(plugin)` (throws on duplicate `id`), `all()`, `routes()` (flat), `navItems()` (flat, sorted by `order ?? 100` then `label`), `panels(slot)` (filtered + sorted by `order ?? 100`).

### `FabriqAdmin.tsx`
- `FabriqAdminProps`: `client, plugins, theme?, basePath?, initialPath?, queryClient?`.
- `FabriqAdmin` component:
  - Builds `PluginRegistry` from `plugins` (memoized on `plugins` identity).
  - Wraps in `<FabriqProvider client queryClient>` — supplies react-query context.
  - Renders `<div className="fabriq-admin" data-fabriq-theme={theme ?? "system"}>` as scoped container.
  - Empty state: "No plugins loaded" when no plugins registered.
  - `<nav>` with buttons per `registry.navItems()` calling `navigate(item.to)`.
  - `<main>` renders matched route's `element` with `{ params }`, or "Not found".
- `PluginHostContext` + `usePluginHost()` exported for panels/plugins to access `{ registry, navigate, path }`.

### `index.ts` additions
Re-exports: `matchRoute`, `useInternalRouter`, `RouterState`, `RouteMatch`, `PluginRegistry`, `FabriqAdmin`, `FabriqAdminProps`, `usePluginHost`, `PluginHostContext`, `PluginHostValue`.

## Embeddability approach
- No `BrowserRouter` or `HashRouter`. Internal router is pure `useState`.
- No `<html>`, `<body>`, no global CSS reset, no document-level styles.
- Scoped container `div.fabriq-admin[data-fabriq-theme]` isolates styling.
- No top-level `window`/`document` access anywhere — SSR-safe to import in Next.js pages.
- `<FabriqProvider>` creates its own `QueryClient` if none is supplied — host apps can inject their own.
