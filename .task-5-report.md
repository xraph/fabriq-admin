# Task 5 Report — Entity Browser Builtin Plugin

## Plugin shape

Package: `@fabriq/plugin-entity-browser` at `plugins/entity-browser/`

```ts
entityBrowserPlugin = definePlugin({
  id: "fabriq.entity-browser",
  name: "Entities",
  version: "0.0.0",
  capabilities: ["entities.read"],
  navItems: [{ label: "Entities", to: "entities", order: 10 }],
  routes: [
    { path: "entities",     element: EntityList,   title: "Entities" },
    { path: "entities/:id", element: EntityDetail, title: "Entity"   },
  ],
})
```

Components:
- **EntityList** — `useFabriqQuery(["entities", type], c => c.listEntities({type}))`, renders a `<table>` of id+type rows, each row clickable → navigates to `entities/<id>`. Includes a text input for type filtering.
- **EntityDetail** — `useFabriqQuery(["entity", id], c => c.getEntity(id))`, renders id, type, and `<pre>` JSON of `data`. Has a Back button → navigates to `entities`.

## SDK exports added

`packages/admin-sdk/src/provider.tsx` — added `export { QueryClient }` so plugin authors (and tests) can configure a `QueryClient` without reaching into `@tanstack/react-query` directly.
