## API settled on

### `packages/admin-sdk/src/client.ts`

**`FabriqTransport` interface** — injectable seam, two methods:
- `request<T>(opts)` — `method?`, `path`, `query?`, `signal?` → `Promise<T>`
- `stream(opts)` — `path`, `body?`, `signal?` → `AsyncIterable<unknown>`

**`FabriqClient`** — constructed with `{ baseUrl, transport }`. `baseUrl` trailing slash is trimmed on construction. Methods:
- `getMeta()` → GET `${baseUrl}/meta`
- `listEntities(params?)` → GET `${baseUrl}/entities` — query is only forwarded when params are present and non-undefined
- `getEntity(id)` → GET `${baseUrl}/entities/${encodeURIComponent(id)}`
- `watch(scope)` → `transport.stream({ path: '${baseUrl}/watch', body: scope })`

**Domain types**: `AdminMeta`, `EntityRecord`, `EntityPage`, `WatchScope` — minimal, all exported.

### `packages/admin-sdk/src/provider.tsx`

- `FabriqProvider` — props `{ client, queryClient?, children }`. Creates a `QueryClient` via `useMemo` if none passed; wraps in `FabriqClientContext.Provider` + `QueryClientProvider`.
- `useFabriqClient()` — reads context; throws `"useFabriqClient must be used within <FabriqProvider>"` if null.
- `useFabriqQuery<T>(key, selector, options?)` — `selector: (client: FabriqClient) => Promise<T>`; thin wrapper over react-query `useQuery` injecting the client. Returns full `UseQueryResult<T, Error>`.
