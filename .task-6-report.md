## Part A — HTTP Transport

### API

`createHttpTransport({ baseUrl, fetchImpl?, headers? }): FabriqTransport`

Exported from `packages/admin-sdk/src/httpTransport.ts` and re-exported from `src/index.ts`.

**`request`**: Joins baseUrl + path, appends URLSearchParams (skips undefined values), calls fetch with method/headers/signal, throws `HttpTransportError(status, body)` on `!res.ok`, returns `res.json()`.

**`stream`**: POSTs body as JSON, reads SSE response body as ReadableStream via `getReader()`. Maintains a running buffer, splits on `\n\n` for event boundaries, filters `data:` prefixed lines, parses JSON. Handles chunks spanning multiple `enqueue` calls via the buffer. Abort via `signal` passed to fetch.

## Part B — Remote Plugin Loader + assertValidPlugin

### `loadRemotePlugin` design

`loadRemotePlugin(opts: RemotePluginOptions): Promise<FabriqAdminPlugin>`

Located at `packages/admin-sdk/src/remoteLoader.ts`.

**Module Federation runtime loading flow:**
1. `loadEntry(url)` — injects a `<script>` tag (default browser impl) or calls a custom injector. Resolves when the script fires `onload`.
2. `getContainer(scope)` — reads `window[scope]` (default) or uses a custom resolver. Returns a `RemoteContainer`.
3. `container.init({})` — shared-scope handshake if present (MF singleton module negotiation). We pass `{}` as the shared scope; a production Webpack host would pass `__webpack_share_scopes__.default`.
4. `container.get(module)` → async factory → `factory()` → plugin module.
5. Module resolution priority: `.default` → `.[moduleBasename]` (from `"./entityBrowserPlugin"` → `"entityBrowserPlugin"`) → the module itself.
6. `assertValidPlugin(candidate)` — throws if id/name/version missing/empty.

**Testing seam**: Both `loadEntry` and `getContainer` are fully injectable. Tests pass fake containers directly — no `window`, `document`, or real scripts touched. Default implementations are browser-only and are only accessed inside function bodies (never at module scope), making the module safe to import in Node.

**No-op loadEntry optimization**: when a custom `getContainer` is provided without an explicit `loadEntry`, `loadEntry` defaults to `async () => {}` — avoids the jsdom script injection hang in tests.

**`assertValidPlugin`**: exported separately for reuse. `definePlugin` in `plugin.ts` keeps inline validation (same checks, no circular import), with a doc comment pointing to `assertValidPlugin` as the canonical source of truth.

**TDD cycle**: 10 test cases written first → failed → implemented → all 10 pass.
